# salsaAuto 更新日志

## v2.1.0 (2026-05-14)

### 新功能

#### 1. 刷单计算器

**位置**: 计算器页面

**功能**: 根据总金额、日期范围、商品配比，自动计算每日刷单方案

**核心算法**:
- 高价商品（≥15元）数量平衡：每天数量相近，最多 20% 差距
- 贪心算法：从贵到便宜填满每日目标金额
- 精度处理：所有金额转为"分"计算，避免浮点数精度问题
- 最后一天特殊处理：重新计算剩余金额，确保完美匹配

**输入参数**:
- 门店选择
- 支付方式（会员支付/现金支付）
- 开始日期、结束日期
- 总金额
- 每日浮动百分比
- 商品选择（可多选）

**输出**:
- 每日商品数量分配表
- 导出 CSV 功能

#### 2. 任务系统

**任务保存**:
- 计算器计算完成后，可保存为任务 JSON 文件
- 任务文件保存在 `data/tasks/` 目录
- 任务命名格式：`{门店}_{支付方式}_{日期区间}_{时间戳}`

**任务结构**:
```json
{
  "task_id": "task_20260514_173100",
  "task_name": "丰台店_会员支付_20260101-20260131_173100",
  "created_at": "2026-05-14T17:31:00",
  "store_id": "13",
  "store_name": "丰台店",
  "pay_type": "memberPay",
  "pay_type_name": "会员支付",
  "total_amount": 10000.00,
  "date_range": {
    "start": "2026-01-01",
    "end": "2026-01-31"
  },
  "days": [
    {
      "date": "2026-01-01",
      "dishes": [
        {"id": "28", "name": "自助餐", "price": 17.6, "quantity": 5}
      ],
      "total_amount": 147.70
    }
  ]
}
```

**任务刷单**:
- 位置：订单页面 → 任务刷单 Tab
- 选择任务后显示详情
- 支持会员类型选择（通用/一纸满）
- 执行时自动：
  - 从商品配置获取可用时间范围
  - 拼接日期 + 时间上界作为交易时间
  - 按任务配置执行刷单

#### 3. 商品时间配置

**配置位置**: `config/settings.json` → `dishes`

**新增字段**: `time_range`

**示例**:
```json
{
  "28": {
    "name": "自助餐",
    "price": 17.6,
    "time_range": "11:00-20:00"
  },
  "77": {
    "name": "烤肉",
    "price": 49.9,
    "time_range": "16:00-23:00"
  }
}
```

**用途**: 任务刷单时，取时间上界作为交易时间

### API 新增

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tasks/save` | 保存任务 |
| GET | `/api/tasks/list` | 获取任务列表 |
| GET | `/api/tasks/{task_id}` | 获取任务详情 |
| POST | `/api/tasks/{task_id}/execute` | 执行任务刷单 |

### 技术要点

1. **浮点数精度处理**: 所有金额转为整数"分"计算，最后转回元
2. **最后一天精确匹配**: 用总金额减去已分配金额，避免精度累积
3. **高价商品平衡**: 每天分配相近数量，模拟真实销售
4. **Token 自动加载**: 服务启动时自动加载已保存的 Token
