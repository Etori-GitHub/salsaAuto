## v2.8.0 (2026-06-04)

### 重构：库存管理系统

按主人的设计重构了整个库存管理系统，解决之前「查询流水为0」的问题。

#### 核心改进

**数据流重构**：
```
旧方案：每次查询调API → 实时计算 → 结果不稳定
新方案：同步到本地缓存 → 查询本地表 → 结果稳定快速
```

**新增数据同步机制**：
- 手动同步按钮
- 同步全部要货明细到本地表 `supply_order_details`
- 同步全部耗用记录到本地表 `consume_records`
- 同步后数据稳定，不因API变化影响历史查询

**库存流水展示**：
- 合并要货（入库）和耗用（出库）数据
- 按时间排序，显示每条记录的库存余额
- 入库显示绿色 `+`，出库显示红色 `-`

**库存查询页面**（新增）：
- 查询当前库存（按门店/商品筛选）
- 显示总数量和总金额
- 点击「流水」按钮跳转到该商品的流水查询

#### 新增文件

| 文件 | 功能 |
|------|------|
| `src/services/data_sync.py` | 数据同步服务 |
| `src/services/stock_flow.py` | 库存流水服务（重构）|

#### 新增数据库表

## v2.5.1 (2026-05-25)

### 补采购功能优化

**问题修复**:
1. 新增排除 G00003、G00004 供应商的商品
2. 优化贪心算法，改为两阶段贪心：先用大金额商品（>1元）填满大部分，再用小金额商品（≤1元）精确补齐零头
3. 新增验算步骤，确保最终金额准确
4. 新增按供应商分组处理，优化采购订单创建流程

**补采购流程**:
1. 读取任务配置（汇总主体、目标金额、日期等）
2. 同步采购明细到本地库
3. 查询指定日期、汇总主体、显示状态的采购明细总金额
4. 计算差额（目标金额 - 当前金额）
5. 如果差额 < 0：关闭部分明细（从最接近差额的开始）
6. 如果差额 > 0：贪心算法选择商品
   - 只使用贸易品（排除现采、加工品）
   - 排除 G00003、G00004 供应商
   - 只取上架状态的商品
   - 两阶段贪心：大金额商品填大部分，小金额商品（0.1元、0.01元）补零头
7. 按供应商分组，查找/创建采购订单
8. 添加采购明细并入库
9. 验算最终金额，返回结果

**贪心算法优化**:
- 第一阶段：用大金额商品（>1元）填满大部分差额
- 第二阶段：用小金额商品（≤1元，如 1元、0.1元、0.01元）精确补齐零头
- 第三阶段：如有极小剩余（<0.005），用最小价格商品补一个
- 可精确补齐到 0.01 元级别

**日志增强**:
- 打印供应商分组情况
- 打印每个供应商的处理过程
- 打印最终验算结果

### 技术要点

1. **贪心算法选商品**：从大到小选择商品，用最小价格商品补齐差额
2. **汇总主体分组**：供应商按 summary_entity 分组，补采购时只使用同一汇总主体下的商品
3. **排除现采/加工品**：补采购只使用贸易品（排除现采和加工品类商品）
4. **自动创建采购单**：如当天该供应商无采购单，自动创建
5. **同步本地库**：执行补采购前先同步采购明细到本地库，提高查询效率

## v2.1.0 (2026-05-14)

### 新功能

#### 1. 刷单计算器

**位置**: 计算器页面

**功能**: 根据总金额、日期范围、商品配比，自动计算每日刷单方案

**核心算法**:
- 高价商品（≥15元）数量平衡：每天数量相近，最多 20% 差距
- 贪心算法：从贵到便宜填满每日目标金额
- 精度处理：所有金额转为"分"计算，避免浮点数精度问题
- 最后一天特殊处理：重新计算剩余金额，确保完美匹配

**输入参数**:
- 门店选择
- 支付方式（会员支付/现金支付）
- 开始日期、结束日期
- 总金额
- 每日浮动百分比
- 商品选择（可多选）

**输出**:
- 每日商品数量分配表
- 导出 CSV 功能

#### 2. 任务系统

**任务保存**:
- 计算器计算完成后，可保存为任务 JSON 文件
- 任务文件保存在 `data/tasks/` 目录
- 任务命名格式：`{门店}_{支付方式}_{日期区间}_{时间戳}`

**任务结构**:
```json
{
  "task_id": "task_20260514_173100",
  "task_name": "丰台店_会员支付_20260101-20260131_173100",
  "created_at": "2026-05-14T17:31:00",
  "store_id": "13",
  "store_name": "丰台店",
  "pay_type": "memberPay",
  "pay_type_name": "会员支付",
  "total_amount": 10000.00,
  "date_range": {
    "start": "2026-01-01",
    "end": "2026-01-31"
  },
  "days": [
    {
      "date": "2026-01-01",
      "dishes": [
        {"id": "28", "name": "自助餐", "price": 17.6, "quantity": 5}
      ],
      "total_amount": 147.70
    }
  ]
}
```

**任务刷单**:
- 位置：订单页面 → 任务刷单 Tab
- 选择任务后显示详情
- 支持会员类型选择（通用/一纸满）
- 执行时自动：
  - 从商品配置获取可用时间范围
  - 拼接日期 + 时间上界作为交易时间
  - 按任务配置执行刷单

#### 3. 商品时间配置

**配置位置**: `config/settings.json` → `dishes`

**新增字段**: `time_range`

**示例**:
```json

  }
```

**用途**: 任务刷单时，取时间上界作为交易时间
