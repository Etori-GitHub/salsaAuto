# 采购任务与补采购开发方案

## 一、采购任务（生成任务）

### 1.1 输入参数
| 参数 | 类型 | 说明 |
|------|------|------|
| 汇总主体 | 下拉选择 | 从 summary_entities 表获取 |
| 时间区间 | 日期选择 | 开始日期、结束日期 |
| 采购总金额 | 数字 | 目标总金额 |
| 每日浮动百分比 | 数字 | 如 10%，每天金额可在平均值上下浮动 |
| 采购人 | 文本 | 用于采购订单的 purchaser 字段 |

### 1.2 计算逻辑
1. 计算天数 N = 结束日期 - 开始日期 + 1
2. 计算平均金额 = 总金额 / N
3. 计算浮动范围 = 平均金额 × 浮动百分比
4. 为每天生成目标金额（在浮动范围内随机）
5. 为每天随机生成一个时间（如 10:23:45）

### 1.3 输出格式
```json
{
  "task_id": "purchase_20260522_154900",
  "summary_entity": "外部供应商",
  "total_amount": 10000,
  "daily_float_percent": 10,
  "purchaser": "system",
  "start_date": "2026-05-22",
  "end_date": "2026-05-31",
  "days": [
    {
      "date": "2026-05-22",
      "target_amount": 1023.45,
      "purchase_time": "2026-05-22 14:23:45"
    },
    ...
  ],
  "created_at": "2026-05-22 15:49:00"
}
```

### 1.4 文件存储
- 目录：`data/purchase-tasks/`
- 文件名：`{汇总主体}_{日期区间}_{时间戳}.json`

## 二、补采购（执行任务）

### 2.1 执行流程

```
1. 读取任务文件
2. 选择要执行的日期
3. 获取当天的采购明细
   - 筛选条件：时间=当天，汇总主体=任务指定，canShow=1
4. 计算当前总金额 C
5. 计算差额 D = 目标金额 - C
6. 调整策略：
   - D < 0：关闭明细，从最接近 |D| 的开始
   - D > 0：增加采购
7. 增加采购流程：
   a. 用贪心算法选择商品组合
   b. 按供应商分组
   c. 为每个供应商创建采购订单
   d. 创建采购明细
   e. 创建入库明细
8. 汇总检查
```

## 六、数据关联

```
汇总主体 (summary_entities)
    ↓
供应商 (suppliers.summary_entity)
    ↓
商品 (goods.supplierCode → suppliers.supplier_code)
```

通过这个关联链，可以筛选出属于某个汇总主体的所有商品。
